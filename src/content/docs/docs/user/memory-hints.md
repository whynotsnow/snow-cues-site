---
title: 记忆提示
description: 关键密钥记忆提示如何加密保存、查看、编辑，以及它的安全边界。
---

记忆提示用于帮助用户回忆关键密钥，但它不是关键密钥本身。它是一段加密保存的辅助提示文字。

## 基本性质

- 记忆提示**会加密保存**，写入 `encrypted_memory_hint` 字段。
- 它**不参与密码生成**。
- 它**不参与密码解密**。
- 它**不能用于自动恢复关键密钥**——系统不会解析它，也不会用它重建任何秘密。

## 查看

查看记忆提示只依赖当前空间会话能力，**不需要再次输入关键密钥**。

- 若条目保存了记忆提示，列表中会展示「查看记忆提示」按钮。
- 点击后解密并展示提示，再点击「隐藏记忆提示」收起。
- 空间校验未完成时，只允许在空间主页查看当前待校验条目的提示。

## 编辑

记忆提示不常驻编辑。编辑条目时该字段默认隐藏并显示掩码：

- 点击「显示提示」只读查看。
- 点击「隐藏提示」收起并清除编辑态明文。
- 点击「解锁编辑 / 添加提示」后才可编辑。

保存时重新加密写入 `encrypted_memory_hint`。

## 清除提示

清除提示必须通过明确的「清除提示」按钮完成，不要用空输入隐式删除。这样可以避免把空输入误当作删除。

## 安全边界

记忆提示会加密保存，但它仍是敏感辅助信息。

:::warning
不要把完整关键密钥、账号密码、恢复口令或其他敏感秘密写进记忆提示。建议只写自己能理解、别人难以利用的提醒。
:::

不同空间主密码、空间或条目之间都不能互相解密提示。提示使用加密保存，查看和编辑都必须由用户显式触发。

## 解密失败怎么办

查看提示后仍无法解密密码时，界面会建议你重置平台密码，并将当前条目标记为废弃。系统**不会自动废弃**。

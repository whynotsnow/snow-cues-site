## 基本概念

### Snow Cues 是密码管理器吗？

Snow Cues 是一个本地优先的密码派生与加密存储系统。它可以保存加密后的密码输出和管理元数据，但不提供云端账号、团队共享、浏览器扩展自动填充或服务商托管密码库。

### 什么是 `storageData`？

`storageData` 是 Snow Cues 的业务数据文件夹。它保存当前数据、历史 revision、草稿和冲突候选。Snow Cues 只把用户显式打开或新建的 `storageData` 作为业务数据源。

### 什么是空间？

空间是 `storageData` 中的本地分区。你可以为不同用途创建不同空间。空间 ID 用于分区和展示，不参与单条密码派生。

### 什么是空间主密码？

空间主密码用于建立当前浏览器内存会话。它不会保存到 `storageData`，也不会作为可找回材料保存。

### 什么是关键密钥？

关键密钥是每条密码单独需要的输入。新建密码和解密密码都必须输入关键密钥。Snow Cues 不保存关键密钥。

## 安全与隐私

### 我的数据会上传到官网吗？

不会。正式推荐的 Cloudflare Pages 只分发静态前端代码。你的 `storageData`、主密码、关键密钥和密码内容不会上传给 Snow Cues 官网。

### 忘记空间主密码或关键密钥怎么办？

Snow Cues 没有找回后门。如果忘记空间主密码或关键密钥，应用没有保存可用于恢复它们的材料。建议自行建立可靠的离线保管策略。

忘记某条密码的关键密钥时，可以先查看该条目的记忆提示。如果仍无法想起，需要去外部平台重置密码，并将旧条目标记为废弃。

### 记忆提示安全吗？

记忆提示会加密保存，但仍应视为敏感辅助信息。不要把完整关键密钥、账号密码或恢复口令写入提示。

### 平台和备注会加密吗？

平台、密码组名称和普通备注是明文管理元数据。不要在这些字段中填写账号、关键密钥或其他敏感秘密。

## 运行环境

### 为什么还推荐 HTTPS 官网？

移动端浏览器通常要求安全上下文才开放完整 WebCrypto 能力。HTTPS 官网可以稳定加载前端应用，同时不承载用户业务数据。

### 可以直接打开本地 HTML 使用吗？

不推荐。直接打开 `file://` HTML 可能导致 WebCrypto、ES module 或文件夹访问能力不可用。正式使用建议通过 HTTPS 地址或 PWA，本地开发使用 `localhost` 或 `127.0.0.1`。

### 普通局域网 HTTP 可以用吗？

不应作为可靠正式环境。`http://192.168.x.x` 这类地址在移动端可能缺少 `crypto.subtle`，导致核心密码学能力不可用。

## 数据保存

### Snow Cues 会自动保存吗？

不会。业务操作先进入浏览器内存草稿。保存必须由用户显式触发，并在保存前确认摘要 diff。

### 为什么进入空间后还要校验？

Snow Cues 不保存校验材料。已有密码的空间进入后，需要通过解密一条已知关键密钥的密码来确认当前会话能力。

### 如果同步工具同时改了文件怎么办？

保存时 Snow Cues 会重读 `current.json` 并检查 revision / hash。如果发现外部变化，会拒绝覆盖，并把本次待保存结果写入 `conflicts/`。

### 换浏览器后为什么看不到空间？

空间数据保存在你显式打开的 `storageData` 中，不在浏览器本地账号里。换浏览器、换设备或清理数据后，需要打开原来的 `storageData` 文件夹或导入备份恢复。

## 密码生成

### 密码组有什么用？

密码组用于归类同一平台或系统下的多个账号，并可保存默认输出适配策略。它不参与核心密码生成或解密。

### 输出适配会改变已保存的密码吗？

不会。输出适配只在核心密码解密后临时应用，不覆盖已保存的核心密码密文。

### 可以导入自定义规则吗？

可以导入声明式 JSON 规则，但不能导入或执行 JavaScript 代码。导入规则只能映射到允许的内置算法模板。

## 迁移与同步

### Snow Cues 支持云同步吗？

应用本身不提供云同步。你可以使用 Syncthing、Git 或其他外部工具管理 `storageData` 文件夹，但同步和冲突合并不属于 Snow Cues 应用逻辑。

### 可以用 Syncthing 同步吗？

可以。推荐同步完整 `storageData` 文件夹，而不是只同步单个 `current.json`。首次使用时，先在一台主设备完成创建、保存和备份，再把 `storageData` 放入 Syncthing 同步目录。

详细流程见：[多设备同步](/docs/user/multi-device-sync/)。

### 同步冲突怎么办？

如果 Snow Cues 保存时发现 `current.json` 已被外部工具修改，会拒绝覆盖，并把当前待保存结果写入 `conflicts/`。不要直接删除冲突文件，先确认各设备同步状态，再比较当前文件和冲突候选。

详细说明见：[存储数据与保存](/docs/user/storage-data/) 和 [多设备同步](/docs/user/multi-device-sync/)。

### 什么是游离密码？

游离密码是系统工具中的空间外临时预览功能。它不保存派生输入、预览结果或历史，也不会写入迁移队列。

### 迁移会直接改掉旧空间吗？

迁移条目会先进入独立迁移队列。全部迁移完成后，系统可以创建接替关系并把来源空间流转为历史空间。这个关系只用于展示，不跨空间继承密钥或密码条目。

### 可以复用关键密钥吗？

Snow Cues 不会阻止你复用相同或相似的关键密钥。这样可以降低记忆负担，但一个关键密钥泄露后，相关条目的风险会一起上升。高价值账户建议使用更独立的关键密钥。

详细取舍见：[安全性与便利性的取舍](/docs/user/security-convenience-tradeoffs/)。

### 应用内指引为什么让我看文档？

应用内指引适合提示下一步操作，但有些问题需要理解背景、风险或外部工具协作方式，例如多设备同步、备份策略、迁移流程或安全性与便利性的取舍。

如果你想理解指引本身，见：[应用内指引](/docs/user/in-app-guidance/)。

## 故障处理

### 解密失败怎么办？

通常需要检查空间主密码、关键密钥或是否进入了正确空间。若该条目保存了记忆提示，可以先查看提示后重试。查看提示后仍失败时，建议去外部平台重置密码，并将当前条目标记为废弃。

### 历史空间和归档空间有什么区别？

历史空间保留必要的历史查看和解密能力，但冻结创建、编辑和规则变更。归档空间用于封存，不支持密码解密或日常派生。

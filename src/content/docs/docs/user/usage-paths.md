这篇文档帮助你决定下一步应该读什么、做什么。它不是按钮说明，而是按使用场景把 Snow Cues 的关键流程串起来。

如果你是从应用内指引跳转过来的，可以先根据当前目标选择对应路径，再进入更详细的专题页面。

## 第一次使用

第一次使用时，目标是创建一份可保存、可备份、以后能重新打开的 `storageData`。

推荐顺序：

1. 阅读 [快速开始](/docs/user/quick-start/)，完成可信运行环境、新建 `storageData`、进入空间、设置空间主密码和创建第一条密码。
2. 阅读 [核心概念](/docs/user/core-concepts/)，确认自己理解空间主密码、关键密钥、记忆提示和规则链的区别。
3. 阅读 [存储数据与保存](/docs/user/storage-data/)，理解显式保存、revision 和冲突候选。
4. 创建一份离线备份，并确认以后能重新打开。

:::warning
不要只完成创建密码这一步。Snow Cues 的业务操作会先进入浏览器内存草稿，保存必须由你显式触发。
:::

## 单设备本地使用

如果你只在一台设备上使用 Snow Cues，重点是稳定保存和定期备份。

推荐流程：

1. 把 `storageData` 放在你能长期管理的位置。
2. 每次修改后点击「保存存储数据」。
3. 定期导出或复制完整 `storageData`。
4. 在清理浏览器数据、重装系统或更换设备前，先确认备份可用。

继续阅读：[存储数据与保存](/docs/user/storage-data/)。

## 多设备使用

如果你希望在多台设备之间使用同一份数据，Snow Cues 本身不提供云同步。推荐使用 Syncthing 等外部工具同步 `storageData` 文件夹。

推荐顺序：

1. 先在主设备创建并保存 `storageData`。
2. 再将 `storageData` 放入 Syncthing 同步目录。
3. 等其他设备同步完成后，在 Snow Cues 中打开同一份数据。
4. 避免多台设备同时编辑同一份 `storageData`。
5. 保留独立备份，不要把同步当作备份。

继续阅读：[多设备同步](/docs/user/multi-device-sync/)。

## 迁移到新设备或新空间

如果你要换设备、换浏览器、调整空间主密码或迁移密码条目，先区分两类操作：

- **设备迁移**：把同一份 `storageData` 带到新设备，并用原空间主密码和关键密钥继续使用。
- **空间迁移**：创建新空间，通过迁移队列把条目转入新空间，可能涉及新空间主密码、新规则链或新关键密钥。

继续阅读：[克隆、导入与迁移](/docs/user/import-export-migration/)。

## 减少记忆负担

Snow Cues 允许你为了记忆便利性主动做一些取舍，例如复用相同或相似的关键密钥、让记忆提示更直接，或让相似网站使用相似规则。

这些做法不会让 Snow Cues 停止工作，但会改变你的安全边界。你需要知道风险来自哪里，再决定是否接受。

继续阅读：[安全性与便利性的取舍](/docs/user/security-convenience-tradeoffs/)。

## 对应用内指引有疑问

应用内指引用于根据当前状态提示可能的下一步，但它不会替你做安全判断，也不会替你恢复主密码或关键密钥。

如果你不理解为什么出现某条提示、为什么有些状态没有提示，或为什么应用把你带到文档页，继续阅读：[常见问题](/docs/user/faq/) 中的「应用内指引」分类。

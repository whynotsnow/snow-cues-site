---
title: 安全性与便利性的取舍
description: 了解复用关键密钥、简化记忆提示和相似规则对 Snow Cues 安全边界的影响。
---

Snow Cues 不强迫你始终选择最高安全性的使用方式。你可以为了记忆便利性主动做取舍，但这些取舍应该是知情的。

系统仍会按规则派生密码、加密保存数据，并避免保存空间主密码和关键密钥；但它不能抵消弱输入、重复输入或过度直接的提示带来的风险。

## 可以主动取舍的地方

常见便利性取舍包括：

- 多个条目使用相同或相似的关键密钥。
- 使用更容易记住的关键密钥。
- 把记忆提示写得更直接。
- 相似网站使用相似的密码组或输出适配。
- 在迁移时复用旧关键密钥，减少重新记忆的负担。

这些做法不一定完全错误，但会让相关条目的风险更容易互相影响。

## 复用关键密钥的影响

关键密钥是单条密码创建和解密时必须输入的材料。Snow Cues 不保存关键密钥，也不保存可恢复关键密钥的结构化字段。

如果多个条目使用相同或相似关键密钥：

- 你需要记住的内容会减少。
- 一个关键密钥被别人知道后，相关条目的风险会一起上升。
- 如果平台密码泄露，攻击者可能更容易猜测你在其他条目上的输入习惯。
- 高价值账户和低价值账户之间的风险边界会变弱。

更稳妥的做法是：高价值账户使用独立关键密钥；低价值、低影响的场景可以按自己的风险承受能力做有限复用。

## 记忆提示的影响

记忆提示会加密保存，但它仍然是敏感辅助信息。它的目标是帮助你回忆关键密钥，而不是替你保存关键密钥。

更直接的提示会降低记忆负担，也会增加被猜出的可能性。

不推荐写入：

- 完整关键密钥。
- 账号密码或恢复口令。
- 与关键密钥只有轻微改写的文本。
- 其他人也能理解的明显线索。

可以考虑写入：

- 只有你自己能理解的场景线索。
- 指向某段个人记忆的短提示。
- 不直接暴露关键密钥结构的分类提示。

## 主密码、关键密钥和提示不要过度关联

空间主密码、关键密钥和记忆提示承担不同角色。它们之间如果存在明显关联，整体抗猜测能力会下降。

尤其不推荐：

- 空间主密码和关键密钥完全相同。
- 关键密钥只是空间主密码的简单变体。
- 记忆提示直接描述空间主密码或关键密钥。
- 所有条目都使用同一种可被外人理解的命名规律。

Snow Cues 可以避免保存这些秘密，但无法阻止别人根据你的输入习惯进行猜测。

## 相似规则和输出适配的影响

密码组和输出适配用于满足不同平台的密码格式要求。它们不替代关键密钥，也不应该被当作主要安全来源。

如果多个平台使用非常相似的规则和输出习惯，一个平台的密码暴露后，其他平台的密码模式可能更容易被推测。对重要账户，应尽量让关键密钥和使用策略有更清晰的区分。

## 如何选择

更偏安全的选择：

- 空间主密码独立且不与关键密钥相似。
- 高价值账户使用独立关键密钥。
- 记忆提示只写自己能理解的线索。
- 迁移、批量修改或同步前先备份。

更偏便利的选择：

- 对低风险条目有限复用关键密钥。
- 对相似场景使用相似的记忆方式。
- 在迁移时保留部分旧关键密钥，降低重新记忆成本。

不推荐的选择：

- 把真实密码或完整关键密钥写进记忆提示。
- 所有账户使用完全相同的关键密钥。
- 空间主密码和关键密钥相同。
- 把 `storageData`、主密码、关键密钥和提示放在同一个不受保护的位置。

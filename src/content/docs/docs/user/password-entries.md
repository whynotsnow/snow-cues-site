---
title: 密码条目
description: 新建、解密、编辑、查看提示和废弃密码条目的完整操作说明。
---

密码条目保存的是加密后的核心密码输出，以及平台、密码组归属、普通备注等非敏感管理信息。

## 新建密码

创建密码需要填写：

- **平台**：用于识别这条密码。
- **密码组**：可选，用于归类同一平台或系统的多个账号。
- **普通备注**：可选，只作为明文管理信息。
- **关键密钥**：必填，用于生成并加密这条密码。
- **关键密钥记忆提示**：可选，会加密保存。

创建成功后，Snow Cues 会清空创建表单中的敏感输入。新密码进入内存草稿，仍需要你手动保存 `storageData`。

:::note
新建密码前未初始化规则链时，需要先前往规则管理初始化规则链。
:::

## 解密密码

解密已有密码时：

1. 找到对应条目。
2. 点击「解密」。
3. 输入该条目的关键密钥。
4. 确认解密。

如果解密失败，通常需要检查空间主密码、关键密钥或是否进入了正确空间。若该条目保存了记忆提示，可以先查看提示后重试。查看提示后仍失败时，建议去外部平台重置密码，并将当前条目标记为废弃。

解密成功后，密码只在当前页面状态中显示。离开空间、会话过期或隐藏密码后，敏感状态会被清除。

## 记忆提示

记忆提示用于帮助回忆关键密钥，但它不是关键密钥本身。提示会加密保存，查看和编辑都需要用户显式操作。

详见[记忆提示](/docs/user/memory-hints/)。

## 密码组与输出适配

密码组用于归类多个账号，并可保存默认输出适配策略。输出适配用于把核心密码转换成满足某些字符长度或字符集要求的临时结果。

注意：

- 输出适配不参与核心密码生成。
- 输出适配不覆盖已保存的核心密码密文。
- 内置预设是常见策略，不代表具体平台官方规则。

详见[密码组与输出适配](/docs/user/password-groups/)。

## 编辑条目

空间校验完成后，可以点击「编辑条目」维护平台、普通备注和记忆提示。

:::note
当前版本允许修改平台和描述，暂不支持直接编辑已保存的密码内容。
:::

## 废弃条目

密码可以标记为废弃。废弃表示该条记录进入历史状态，**不会删除数据**。适合平台密码已经重置或不再使用的情况。

废弃后平台不可修改，描述仍可修改。

## 明文元数据提醒

平台、密码组名称和普通备注是明文管理元数据。它们方便识别条目，但**不应填写账号、关键密钥、密码或其他敏感秘密**。
